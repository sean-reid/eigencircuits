swiftlatexpdftex.js and swiftlatexpdftex.wasm are builds of the SwiftLaTeX
pdfTeX engine. swiftlatexpdftex.js carries one local change: when a fetched
TeX file has no `fileid` response header, it falls back to the requested file
name, so the engine works against a plain static host (our Cloudflare assets)
that does not set that header.

  Source:  https://github.com/SwiftLaTeX/SwiftLaTeX
  Author:  Elliott Wen and contributors
  License: EPL-2.0 OR GPL-2.0 WITH Classpath-exception-2.0

They are bundled here so the PDF compiler runs entirely in the browser from
this origin. TeX packages are fetched on demand from the SwiftLaTeX TeX Live
mirror (https://texlive2.swiftlatex.com/) and cached by the worker.
